## Ways to Contribute | 貢獻方式

### 1. Share Your Examples | 分享您的範例

Have a useful pattern or technique for Claude Code? We'd love to include it!

有實用的 Claude Code 模式或技巧嗎？歡迎分享！

**To contribute an example | 貢獻範例的步驟：**
1. Fork this repository | Fork 這個專案
2. Create a new file in `/examples` or `/concepts` | 在 `/examples` 或 `/concepts` 建立新檔案
3. Follow the existing format | 遵循現有格式
4. Submit a Pull Request | 提交 Pull Request

### 2. Translate Content | 翻譯內容

Help us make content accessible to more developers!

幫助我們讓更多開發者能夠使用這些內容！

**Translation Guidelines | 翻譯指南：**

| Do | Don't |
|---|---|
| Keep technical terms in English (Claude Code, MCP, Hooks, Agent) | 技術術語保持英文 |
| Translate explanations and descriptions | 翻譯說明和描述 |
| Keep code examples unchanged | 程式碼範例保持不變 |
| Use Taiwan Traditional Chinese conventions | 使用台灣繁體中文用語 |
| Maintain markdown formatting | 保持 markdown 格式 |

**Common Terms | 常見用語對照：**

| English | 繁體中文 |
|---------|---------|
| file | 檔案 |
| code | 程式碼 |
| server | 伺服器 |
| database | 資料庫 |
| documentation | 文件 |
| developer | 開發者 |

### 3. Improve Existing Content | 改善現有內容

Found a typo? Have a better explanation? PRs are welcome!

發現錯字？有更好的說明？歡迎提交 PR！

### 4. Join the Discussion | 加入討論

For questions, ideas, and discussions:

如有問題、想法和討論：

**[Join our Discord: Claude World Taiwan](https://discord.gg/rBtHzSD288)**

Discord 是最好的地方：
- Ask questions about Claude Code | 詢問 Claude Code 相關問題
- Share your experiences | 分享您的經驗
- Get help with specific problems | 獲得特定問題的幫助
- Discuss new ideas before implementing | 在實作前討論新想法

## Content Guidelines | 內容指南

### Do Include | 應包含

- Practical, tested techniques | 實用且經過測試的技巧
- Clear explanations with examples | 有範例的清楚說明
- Code snippets that work | 可運作的程式碼片段
- Before/after comparisons | 前後對比

### Don't Include | 不應包含

- Untested or theoretical approaches | 未測試或理論性的方法
- Vendor-specific content (keep it general) | 特定廠商的內容（保持通用性）
- Proprietary code or secrets | 專有代碼或機密
- Content that violates Anthropic's usage policies | 違反 Anthropic 使用政策的內容

---

## Format Guidelines | 格式指南

### File Structure | 檔案結構

```markdown
# Title | 標題

> One-line summary | 一句話摘要

## The Problem | 問題

[Explain what problem this solves]
[說明這解決什麼問題]

## The Solution | 解決方案

[Explain the approach]
[說明方法]

## Example | 範例

[Provide working example]
[提供可運作的範例]

## Best Practices | 最佳實踐

[List dos and don'ts]
[列出該做和不該做的事]

---

See also: [Related Topic](./link.md)
另見：[相關主題](./zh-TW/link.md)
```

### Code Blocks | 程式碼區塊

- Use syntax highlighting | 使用語法高亮
- Keep examples concise | 保持範例簡潔
- Include comments for clarity | 包含註解以增加清晰度
