## Director Mode 決策樹

```
┌─────────────────────────────────────────────────────────────────┐
│                DIRECTOR MODE 決策樹                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  開始：你有一個任務給 Claude                                     │
│           │                                                     │
│           ▼                                                     │
│  ┌─────────────────────┐                                        │
│  │ 結果是否明確定義？   │                                        │
│  └─────────┬───────────┘                                        │
│       是   │  否 → 先定義目標                                    │
│            ▼                                                    │
│  ┌─────────────────────┐                                        │
│  │ 上下文是否在         │                                        │
│  │ CLAUDE.md 中？       │                                        │
│  └─────────┬───────────┘                                        │
│       是   │  否 → 加入 CLAUDE.md                                │
│            ▼                                                    │
│  ┌─────────────────────┐                                        │
│  │ 可以平行執行嗎？     │                                        │
│  └─────────┬───────────┘                                        │
│       是   │  否 → 單一 Agent 即可                               │
│            ▼                                                    │
│  ╔═════════════════════╗                                        │

│  ║                     ║                                        │

│                                                                 │

│  │ ✗ 「你可以幫我...」  → 「實作 X」                         │    │
```

## Parallel Agents 速查表

```
┌─────────────────────────────────────────────────────────────────┐
│                  PARALLEL AGENTS 速查表                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  何時使用 PARALLEL AGENTS：                                      │
│  ┌────────────────────────────────────────────────────────┐     │
│  │ ✓ 探索程式碼庫       │ ✓ Bug 調查                      │     │
│  │ ✓ 程式碼審查         │ ✓ 功能研究                      │     │
│  │ ✓ 多檔案搜尋         │ ✓ 架構分析                      │     │
│  └────────────────────────────────────────────────────────┘     │
│                                                                 │
│  何時不使用：                                                    │
│  ┌────────────────────────────────────────────────────────┐     │
│  │ ✗ 順序相依（A 必須在 B 之前完成）                        │     │
│  │ ✗ 單一檔案編輯                                          │     │
│  │ ✗ 簡單查詢                                              │     │
│  └────────────────────────────────────────────────────────┘     │
│                                                                 │
│  AGENT 類型：                                                   │
│  ┌──────────────┬───────────────────────────────────────┐       │
│  │ Explore      │ 程式碼庫結構、尋找檔案                  │       │
│  │ Grep         │ 搜尋模式/文字                          │       │
│  │ Read         │ 檢視特定檔案                           │       │
│  │ Bash         │ 執行指令、檢查 git                     │       │
│  └──────────────┴───────────────────────────────────────┘       │
│                                                                 │
│  效率計算：                                                      │
│  ┌────────────────────────────────────────────────────────┐     │
│  │  順序執行：5 個任務 × 5 秒 = 25 秒                      │     │
│  │  平行執行：5 個任務同時 = 5 秒                          │     │
│  │                                                        │     │
│  │  加速：獨立任務 5 倍                                    │     │
│  └────────────────────────────────────────────────────────┘     │
│                                                                 │
│  平行模式範例：                                                  │
│  ┌────────────────────────────────────────────────────────┐     │
│  │  「調查驗證 bug」                                       │     │
│  │                                                        │     │
│  │  Agent 1：探索 auth 模組結構                            │     │
│  │  Agent 2：搜尋 "login" 和 "session"                     │     │
│  │  Agent 3：讀取 auth middleware                          │     │
│  │  Agent 4：檢查最近的 auth commits                       │     │
│  │  Agent 5：搜尋錯誤處理                                  │     │
│  │                                                        │     │
│  │  → 全部在約 5 秒內完成                                  │     │
│  │  → 結果自動綜合                                         │     │
│  └────────────────────────────────────────────────────────┘     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## Bug 獵捕工作流程

```
┌─────────────────────────────────────────────────────────────────┐
│                   BUG 獵捕工作流程                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 清楚描述                                                     │
│  ┌────────────────────────────────────────────────────────┐     │
│  │  ✗ 不好：「登入壞了」                                    │     │
│  │  ✓ 好的：「使用者 15 分鐘後登出，預期：活動時              │     │
│  │          session 應該刷新」                              │     │
│  └────────────────────────────────────────────────────────┘     │
│                                                                 │
│  2. 包含上下文                                                   │
│  ┌────────────────────────────────────────────────────────┐     │
│  │  □ 錯誤訊息（確切文字）                                  │     │
│  │  □ 重現步驟                                              │     │
│  │  □ 預期 vs 實際行為                                      │     │
│  │  □ 何時開始（commit？部署？）                            │     │
│  │  □ 已嘗試過的方法                                        │     │
│  └────────────────────────────────────────────────────────┘     │
│                                                                 │
│  3. 讓 CLAUDE 調查                                               │
│  ┌────────────────────────────────────────────────────────┐     │
│  │  Claude 啟動 parallel agents：                           │     │
│  │  → 搜尋錯誤/相關程式碼                                   │     │
│  │  → 檢查最近變更                                          │     │
│  │  → 審查測試                                              │     │
│  │  → 分析模式                                              │     │
│  └────────────────────────────────────────────────────────┘     │
│                                                                 │
│  4. 審查並核准                                                   │
│  ┌────────────────────────────────────────────────────────┐     │
│  │  Claude：「在 /lib/auth.ts:42 發現問題...」              │     │
│  │  你：「修復它」                                          │     │
│  │  Claude：[實作修復 + 測試 + commit]                      │     │
│  └────────────────────────────────────────────────────────┘     │
│                                                                 │
│  典型時間：2-5 分鐘                                              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 功能開發工作流程

```
┌─────────────────────────────────────────────────────────────────┐
│                功能開發工作流程                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  步驟 1：完整定義                                                │
│  ┌────────────────────────────────────────────────────────┐     │
│  │  包含：                                                  │     │
│  │  □ 做什麼（使用者故事）                                  │     │
│  │  □ UI 需求                                               │     │
│  │  □ API 需求                                              │     │
│  │  □ 邊緣案例                                              │     │
│  │  □ 範圍外                                                │     │
│  └────────────────────────────────────────────────────────┘     │
│                                                                 │
│  步驟 2：讓 CLAUDE 分析                                          │
│  ┌────────────────────────────────────────────────────────┐     │
│  │  Claude 研究（平行）：                                    │     │
│  │  → 現有模式                                              │     │
│  │  → 相關程式碼                                            │     │
│  │  → API 慣例                                              │     │
│  │  → 測試模式                                              │     │
│  └────────────────────────────────────────────────────────┘     │
│                                                                 │
│  步驟 3：審查計劃                                                │
│  ┌────────────────────────────────────────────────────────┐     │
│  │  Claude 提議：                                           │     │
│  │  - 資料庫變更                                            │     │
│  │  - 新檔案                                                │     │
│  │  - 修改的檔案                                            │     │
│  │  - 測試                                                  │     │
│  │                                                        │     │
│  │  你：「繼續」或「調整 X」                                 │     │
│  └────────────────────────────────────────────────────────┘     │
│                                                                 │
│  步驟 4：實作與審查                                              │
│  ┌────────────────────────────────────────────────────────┐     │
│  │  Claude 一次實作所有：                                    │     │
│  │  ✓ 所有檔案建立/修改                                     │     │
│  │  ✓ 測試撰寫                                              │     │
│  │  ✓ 測試通過                                              │     │
│  │  → 準備好讓你審查                                        │     │
│  └────────────────────────────────────────────────────────┘     │
│                                                                 │
│  ✗ 不要：漸進式實作（按鈕 → 處理器 → API）                       │
│  ✓ 要：一次完整實作                                             │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 指令快速參考

```
┌─────────────────────────────────────────────────────────────────┐
│                   指令快速參考                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  有效的指令：                                                    │
│  ┌────────────────────────────────────────────────────────┐     │
│  │  「實作 X 並符合 Y 限制」                                 │     │
│  │  「修復 [位置] 的 bug」                                   │     │
│  │  「重構 X 使用 Y 模式」                                   │     │
│  │  「為 X 添加測試，包含邊緣案例」                          │     │
│  │  「調查為何 X 會發生」                                    │     │
│  └────────────────────────────────────────────────────────┘     │
│                                                                 │
│  無效的指令：                                                    │
│  ┌────────────────────────────────────────────────────────┐     │
│  │  「你可以幫我...」        → 直接說                        │     │
│  │  「我在想也許...」        → 明確陳述                      │     │
│  │  「先讀 X，然後讀 Y...」  → 讓 Claude 決定                │     │
│  │  「你覺得...怎麼樣？」    → 給出方向                      │     │
│  └────────────────────────────────────────────────────────┘     │
│                                                                 │
│  加強短語：                                                      │
│  ┌────────────────────────────────────────────────────────┐     │
│  │  「遵循現有模式」                                         │     │
│  │  「完整測試覆蓋」                                         │     │
│  │  「符合專案風格」                                         │     │
│  │  「包含錯誤處理」                                         │     │
│  │  「使用 parallel agents」                                 │     │
│  └────────────────────────────────────────────────────────┘     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
